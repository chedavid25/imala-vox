Confidencial — uso interno y desarrollo
1. Resumen ejecutivo
Imalá Vox es un SaaS multi-tenant de gestión de agentes de inteligencia artificial para WhatsApp Business API, Instagram y Facebook Messenger. Está orientado a profesionales independientes, inmobiliarias y empresas medianas que necesitan automatizar su atención al cliente con IA sin perder el control humano de las conversaciones.
El sistema opera bajo una arquitectura de tres columnas: panel lateral de navegación, bandeja central de operaciones e inbox, y panel derecho de contexto CRM. Técnicamente se construye sobre Next.js, Tailwind CSS, shadcn/ui y Firebase (Firestore + Auth + Storage + Cloud Functions).
OBJETIVO PRINCIPAL: Permitir que cualquier profesional configure un agente de IA que responda en WhatsApp de forma autónoma, con acceso a un catálogo de productos o propiedades, una base de conocimiento personalizada y reglas de negocio específicas — sin conocimientos técnicos.

5. Facturación con MercadoPago
5.1 Estrategia de precios en ARS
Los precios del sistema están denominados en dólares estadounidenses (USD). Los clientes argentinos pagan en pesos argentinos (ARS) a través de MercadoPago. La conversión se realiza utilizando la cotización del dólar blue (informal) más un spread del 10%, actualizado trimestralmente.

API Bluelytics — promedio de últimos 7 días del dólar blue (value_sell)

+10% sobre la cotización blue

Trimestral — enero, abril, julio y octubre

5.2 Flujo técnico de suscripción en MercadoPago
El cliente selecciona plan y ciclo. La Cloud Function calcularPrecioARS() consulta Bluelytics y calcula el monto.
Se crea una suscripción preaprobada con la API de MP Suscripciones con el monto calculado en ARS.
MP envía webhook subscription_authorized_payment al cobrar. La Cloud Function actualiza estado → 'activo' en Firestore.
En caso de fallo: MP reintenta 2 veces. Al segundo fallo: estado → 'pago_vencido'. A los 7 días: estado → 'cancelado'.
En cada ajuste trimestral: se cancela la suscripción MP anterior y se crea una nueva con el precio actualizado. MP no permite modificar el monto de una suscripción existente.
Al mejorar de plan: se cancela la suscripción actual y se crea una nueva con el precio del plan superior al tipo del día.
IMPORTANTE: MercadoPago no prorratea automáticamente en cambios de plan.
Al subir de plan, el cliente paga el nuevo plan completo desde el día del cambio.
Podés compensarlo otorgando días de crédito manualmente desde el SuperAdmin.

7. Módulos del sistema
7.1 OPERACIÓN
Bandeja de entrada (Inbox)
Vista principal de todas las conversaciones activas del workspace.
Filtros por canal (WA / IG / FB), estado (abierto / resuelto / bot), agente asignado y etiquetas.
Indicador visual de estado del agente IA en cada conversación (activo / pausado / bloqueado por contacto Personal).
Transferencia manual de conversación entre agente IA y operador humano.
Composer con soporte para texto, imágenes, documentos y drag-and-drop de objetos del catálogo.
Contactos (CRM híbrido)
Importación desde CSV/Excel y sincronización desde Google Contacts.
Etiqueta de relación obligatoria por contacto: Personal | Laboral | Lead.
Campo aiBlocked: boolean — se activa automáticamente cuando relacionTag === 'Personal'.
El agente IA verifica aiBlocked antes de emitir cualquier respuesta. Si es true, no responde.
Etiquetas de interés personalizables para segmentación de difusiones.
Vista de historial completo de conversaciones por contacto.
Difusión masiva
Creación de audiencias basadas en etiquetas de interés.
Exclusión automática de contactos con relacionTag === 'Personal' — no puede ser desactivada.
Uso de templates aprobados por Meta para mensajes de marketing.
Programación de envíos con fecha y hora.
Estadísticas por campaña: enviados, entregados, leídos, respondidos.
7.2 CEREBRO
Catálogo de objetos
Dos tipos: propiedad (inmueble) y producto (e-commerce o servicio).
Carga dual:
Scraping: el usuario pega una URL (Argenprop, ZonaProp, MercadoLibre) y Puppeteer extrae título, precio, fotos, descripción y características.
Lienzo libre: el usuario arrastra fotos y pega texto — Claude API estructura los datos en campos normalizados.
Campos normalizados: título, precio, descripción, fotos[] (Storage URLs), características{} (m², ambientes, etc.), urlFuente, estado.
La ObjectCard en el panel derecho permite arrastrar el objeto al chat — se envía automáticamente una ficha técnica formateada al cliente.
Base de conocimiento
Carga de archivos PDF, Word y URLs de sitios web como fuente de verdad del agente.
Switch de modo estricto (strictMode): si está activo, el agente solo responde con información de los recursos cargados — no genera respuestas por conocimiento propio.
Soporte para múltiples recursos simultáneos — el agente los consulta como contexto en cada respuesta.
Scraper de sitios web
Gestión de URLs de portales configurados para scraping periódico.
Actualización automática del catálogo cuando se detectan cambios en las propiedades/productos fuente.
7.3 AJUSTES
Configuración del agente IA
Nombre, avatar y personalidad del agente.
Instrucciones de sistema personalizadas (prompt del agente).
Horario de atención — fuera del horario, el agente puede responder un mensaje predefinido o escalar.

Canales

Workflows

Triggers disponibles: mensaje recibido, inactividad N minutos, nueva palabra clave, cambio de etiqueta.
Acciones disponibles: cambiar estado, asignar agente, enviar template, actualizar etiqueta, notificar por email.
Etiquetas con lógica de IA
Etiquetas manuales y etiquetas sugeridas automáticamente por el agente según el contenido de la conversación.

12. Reglas de negocio críticas
Estas reglas NO son opcionales. Deben implementarse desde el inicio y nunca pueden ser desactivadas por el cliente.
Regla
Descripción
Implementación
Exclusión de contactos Personales
El agente IA NUNCA responde a contactos con relacionTag === 'Personal'
Campo aiBlocked en Firestore — verificado en procesarRespuestaIA antes de llamar a Claude API
Exclusión en difusiones
Las difusiones NUNCA incluyen contactos con relacionTag === 'Personal'
Filtro obligatorio en la Cloud Function de difusión — no configurable desde el frontend
Límites de plan
El agente no responde si uso.convCount >= límite del plan
checkLimitesPlantilla verificado antes de cada respuesta del agente
Atomicidad del contador
El incremento de convCount debe ser atómico
Transacción Firestore — no actualización simple — para evitar race conditions
Cotización validada
Nunca aplicar ajuste ARS sin cotización confirmada
Si Bluelytics y dolarapi.com fallan, el ajuste se pospone 24hs y se reintenta
Precio USD como fuente de verdad
precioARS es siempre un campo derivado — nunca la fuente de verdad
Todos los reportes del SuperAdmin trabajan en USD
Webhook Meta verificado
Solo procesar webhooks con X-Hub-Signature-256 válida
Verificación HMAC en la Cloud Function — rechazar si no coincide

13. Filosofía visual
Imalá Vox tiene una identidad visual de herramienta de ingeniería, no de app de consumo. El usuario debe sentir que usa algo serio, rápido y bien construido. Las referencias son Linear.app (densidad + precision) y Vercel (dark mode + tipografía poderosa).
Principio
Descripción

Modo claro total o dark total — el contraste entre zonas es parte del diseño
Un solo acento
El verde lima #C8FF00 es el ÚNICO color de acción primaria. Botones CTA, badges activos, indicadores de IA encendida.
Usar el acento decorativamente o en múltiples elementos a la vez
Bordes finos 1px
